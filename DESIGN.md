## 🧪 コアコンセプト

**「魔法薬の研究室」 × 「ぷるぷるスライムの実験」**

スイカゲーム形式の落下合体パズルに **「混色」** という第二軸を加えた新ジャンル。
パステル調の魔女／錬金術師の工房を舞台に、感情を持つスライム達を育てる。

### コアルール

- 上から液滴を落とす（操作は位置決めのみ）
- **同色同サイズ → 進化**（一段階大きいスライムに）
- **異色同サイズ → 混色**（青 + 黄 = 緑 など）
- ビーカーから溢れたらゲームオーバー

### ゲーム性の軸

> **「進化させたい欲」 vs 「混ざっちゃう恐怖」のジレンマ**

混色は事故にも戦略にもなる。プレイヤーは常に二つの引力の間で揺れる。

### 重要なルール詳細（要検証）

- 接触判定は **同サイズのみ反応**（サイズ違いは事故防止のため反応しない）
- 混色は **Lv1（しずく）同士のみ** で発生 → 上位スライムは混色しない（戦略性確保）
- ※プロト段階で「上位混色も面白い」となれば変更検討

## 🌟 ゲーム要素

### 化学らしいヒネリ（Phase 4 以降）

- **温度ゾーン**: 加熱で流動化、冷却で固化
- **触媒ドロップ**: 反応を加速する特殊液
- **レアレシピ**: 特定順で混ぜると金 / 発光スライム
- **失敗反応**: 相性悪いと煙を上げて減量

### モード

- **エンドレス**: ハイスコア型
- **レシピ**: 「金スライムを作れ」など目標型
- **フリーラボ**: 自由実験、図鑑記録

### 隠し味（Cult of the Lamb 的毒）

- 図鑑コメントが進化ごとに哲学的になる
- 最終進化体（妖精化）が「どこかへ行ってしまう」演出
- 深夜プレイで研究室の照明が変わる

### 世界観

プレイヤーは **見習い魔女 / 錬金術師**。

## ⚙️ 技術スタック

### エンジン: Unity 6 (LTS)

| 領域 | 採用技術 | 用途 |
|---|---|---|
| レンダリング | URP (Universal Render Pipeline) | 2D ライト、Shader Graph |
| 物理 | Rigidbody2D / Collider2D | 標準の 2D 物理 |
| 液体表現 | カスタムシェーダー (HLSL) | メタボール描画。剛体を液体風に |
| アニメーション | DOTween | 可愛い動きの演出 |
| カメラ | Cinemachine | カメラ揺れ・追従 |
| UI | UI Toolkit | HTML/CSS 風の宣言的 UI |
| テキスト | TextMeshPro | 高品質フォント描画 |
| 言語 | C# | Unity 標準 |

### 既存スキルの翻訳

| 持ってるスキル | Unity での活かし方 |
|---|---|
| GLSL / Skia / Shadertoy | **HLSL シェーダー**（構文違うだけ、考え方そのまま） |
| React コンポーネント設計 | **MonoBehaviour** の責務分離 |
| TypeScript 型システム | **C# の型システム**（馴染みやすい） |
| Three.js / WebGL2 | **URP / Shader Graph** |

### 重要な技術判断

> **本物の流体シム (SPH 等) は重すぎ → 剛体 + メタボール描画で液体風に見せる**

- Rigidbody2D で物理計算
- 各球の位置をシェーダーに渡し、距離関数でメタボール合成
- スイカゲームと同じ「合体」表現は剛体の置換でシンプルに

## 📅 開発フェーズ

> 期間は目安、ハマったら一気に進める。納得いくまで磨く。

### Phase 0: Unity キャッチアップ

- Unity Hub セットアップ、2D テンプレートで起動
- 公式 Learn "2D Beginner: Adventure Game" を一周
- Rigidbody2D / Collider2D で球を落として遊ぶ
- Shader Graph で簡単なメタボールを試作
- **完了基準**: 球が落ちて跳ねる WebGL ビルドが出る

### Phase 1: コア触感プロト 🎯 最重要

ここで「面白いか」を判定。**触ってて気持ちいいか**が全て。

- ビーカー 1 つ、1 色のみ
- 球を落とす、同色同サイズで合体（3 段階だけ）
- メタボール描画で液体感
- ぷるぷる Idle アニメ
- **完了基準**: 一人で 5 分触り続けられる

### Phase 2: ジレンマ軸の追加

- 混色ロジック（Lv1 同士の異色 → 新色生成）
- 進化階層を 7 段階に拡張
- 色 × 性格バリエーション
- **完了基準**: 「事故混色した！」「金スライム出た！」が起こる

### Phase 3: 磨き込み（Cult of the Lamb 的毒）

- 演出・アニメーション（見つめ合い、寄り添い眠り、わぷっ）
- 音・BGM
- 図鑑コメントの哲学化
- 妖精化お別れ演出

### Phase 4: 化学要素（余裕があれば）

- 温度・触媒
- レシピ・図鑑システム
- レアレシピ

### Phase 5: WebGL ビルド & リリース

- itch.io 公開
- 反応見て次プラットフォーム判断

## 🚧 既知のリスク・要検証事項

| 項目 | 内容 | 対策 |
|---|---|---|
| アート量 | 7 段階 × 7 色 = 49 スプライト + アニメ | Live2D / Spine + パラメータ展開、AI 生成補助 |
| 混色ルール | 「事故混色だらけ」になる懸念 | 同サイズ判定厳格化、上位は混色しない |
| メタボール負荷 | 球が増えるとシェーダー重くなる | 距離関数の最適化、見える範囲のみ計算 |
| 先行作品との差別化 | スイカゲーム + 混色は類似先行あり | **触感の独自性**（ぷるぷる + メタボール + キャラ）で勝負 |
